Report
Run the corpus to generate an anomaly report
How to read this report
- ok / failed: total successful vs failed executions.
- maxLatencyMs / avgLatencyMs: slowest and average request time.
- anomalies.error: error/exception happened for at least one run.
- anomalies.slow: a run exceeded your Slow ms threshold.
- anomalies.inconsistent: same test returned different signatures across repetitions.
Run with repetitions >= 3 for stronger inconsistency signals.
